Permutation-based Variable Importance for models trained to predict players' wages in the FIFA 23 game¶

We analyse the random forest model and few other models trained in previous assignments on the FIFA 23 game dataset. The model aims to predict players' wages given their statistics in the game.

We calculate Permutation-based Variable Importance for the forest model.

We compare results with Permutation-based Variable Importance for three other models: linear model, log-linear model (linear model with log transformation on the data) and smaller forest model (30 trees instead of 100).

We also compare Permutation-based Variable Importance with Gini-based and SHAP-based Variable Importance for the forest model and analyze the results.

Permutation-based Variable Importance¶

We calculate Permutation-based Variable Importance for the tree model. Since there are more than hundred features, we display only top 5. Those are Overall, Value(in Euro), Country_England, Release Clause and Composure.

We observe that the Overall feature is orders of magnitude more important than other features. Also Value(in Euro) is much more important than all other features but Overall. This suggests strong correlation of this two features with the explained variable (player's wage). Such a correlation is expected and was observed also by other methods during previous assignments.

Comparison with PVI for different models¶

We compare Permutation-based Variable Importance results for four models: linear model, log-linear model and two forest models - one consisting of 100 trees and one consisting of 30 trees.

Both forest models result with similar most important variables. Top 4 are exactly the same, differences appear on the fifth position. It suggests that family of forest models is quite consistent in choosing the most important features. It might also suggest that smaller forest model would be sufficient and using 100 trees is unnecessary. Note that also when comparising different explanation method for the same forest model (the next section) we obtain stability for top 4 features and instability on further positions, which suggests that forest models efficiently choose four most important features.

Most important features for the linear model are different than the ones from the tree models. The only feature that appears both for the linear model and the tree models is Value(in Euro). Similar behaviour was observed during previous assignments - the linear model focuses on different set of features than the tree model.

Quite interesting are results for the log-linear model. Top 3 features are a combination of features chosen by the linear model and the forest models - we have Value(in Euro) which was present in all models, TotalStats that was present only for the linear model and Release_Clause that was present only for the forest models. On the forth and the fifth position the log-linear model introduces its own chosen features. This might suggest that log transformation of the data is quite a good idea as it allows to choose the most important variables in a way that is more consistent with other methods, so more stable.

Comaprison with Gini-based Variable Importance and SHAP-based Variable Importance¶

We compare 5 most important variables for the forest model according to Permutation-based Variable Importance, Gini-based Variable Importance an SHAP-based Variable Importance. We display a few less important features for the SHAP-based Variable Importance method as truncating the plot to Top 5 was technically a bit tedious.

Explanations between these methods are quite similar - top 4 for Permutation-based and SHAP-based Variable Importance are the same and Gini-based Variable Importance swaps third and forth position (Country_England and Release Clause). The explanations differ on the fifth position - each method suggests different feature (Composure, Reactions, International Reputation). This suggest that the four most important features of the model are quite stable as different methods of explanation does not suggest other features for Top 4, the instability appears for the fifth and further features.

Summary¶

We compared PVI for the forest model with Gini-based VI and SHAP-based VI for the same model and with PVI for different models - the linear model, the log-linear model and the smaller forest model.

We observe that the method was quite stable for the tree model - chosen four most important features were consistent with the ones chosen by different VI methods and with chosen by PVI for the smaller forest model.

We also observed that the linear model focuses on different features than the forest models and that the log-linear model combines properties of both approaches which suggests that log transformation of the data is a sensible idea.
